Performance issues remaining

Critical Google Analytics loading 940 KB of unused JS — biggest waste 232 KB wasted

Two GA4 scripts (gtag/js and analytics.js) together load 940 KB but waste 170 KB of unused code. They're also loaded synchronously, adding to your LCP delay. This is your #1 remaining performance drag — GA is heavier than your entire app.

Bundle breakdown
gtag/js (Google Analytics)470 KB · 109 KB wasted
analytics.js (GA tag manager)470 KB · 61 KB wasted
vendor-firebase.js (unminified)535 KB · 28 KB unminified
Load GA with async — it's probably blocking right now. Change to: <script async src="https://www.googletagmanager.com/gtag/js?id=G-2JN5RTZ45Z">
Or load GA only after first user interaction — saves 170 KB on initial load entirely:
// In main.jsx or App.jsx — defer GA until interaction const loadGA = () => { if (window.gaLoaded) return; window.gaLoaded = true; const s = document.createElement('script'); s.src = 'https://www.googletagmanager.com/gtag/js?id=G-2JN5RTZ45Z'; s.async = true; document.head.appendChild(s); }; window.addEventListener('scroll', loadGA, { once: true }); window.addEventListener('click', loadGA, { once: true });
Firebase vendor bundle is partially unminified — run npx vite build and check vite.config.js has build: { minify: 'esbuild' } explicitly set.

High Album art images serving 500×500 px at 36×36 px display size 309 KB wasted

6 song cover images (the small sidebar thumbnails w-9 h-9 = 36×36 px) are fetching 500x500 JPEGs from Saavn CDN. That's 194× more pixels than needed. One image (Humpty-Sharma-Ki-Dulhania) has 0-second cache and wastes 68 KB on every page load.

JioSaavn CDN supports size in URL — replace 500x500 with 50x50 for thumbnails. Add this utility:
// utils/image.js export const saavnImg = (url, size = 50) => url?.replace(/\d+x\d+/, `${size}x${size}`) ?? ''; // Usage in SongCard: <img src={saavnImg(song.image, 50)} alt="Cover" /> // For now-playing vinyl (LCP): <img src={saavnImg(song.image, 300)} fetchpriority="high" alt="Vinyl Label" />
The 0-cache image is a fresh Saavn URL — those images have no cache headers on their end. Cache the URL→blob in IndexedDB or use a service worker to cache responses from c.saavncdn.com.
High Your own JS bundles still only have 1-hour cache 96 KB wasted per revisit

Your Vite-hashed bundles (vendor-firebase-BZH0ZvW1.js, etc.) are safe to cache for 1 year since the hash changes on every build — but Firebase Hosting still serves them with max-age=3600 (1 hour). Every returning user re-downloads your entire app.

This fix takes 2 minutes — add to firebase.json:
{ "hosting": { "headers": [{ "source": "/assets/**", "headers": [{ "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }] }] } }
Medium CSS file is render-blocking — delays first paint 300 ms est. saving

index-CIw1bhKG.css (9 KB) is loaded as a blocking stylesheet. The browser can't paint anything until this file downloads and parses — directly pushing FCP past 1.2 s.

Vite inlines critical CSS automatically in production. Check if your build mode is truly production — run npx vite build --mode production and verify the output.
For the Google Fonts import (if still in CSS): move it to index.html with rel="preload" so it loads in parallel with everything else.

Accessibility issues — score is 74, target 90+

5 issues Accessibility fixes — all in your JSX
Buttons
5 icon buttons have no accessible name
All 5 are button.p-1 in your player controls — play, skip, like etc. Screen readers say "button" with no context. Fix: add aria-label to every icon button.
<!-- Before --> <button className="p-1 rounded bg-[#fff9ef]..."> <PlayIcon /> </button> <!-- After --> <button className="p-1 rounded bg-[#fff9ef]..." aria-label="Play"> <PlayIcon aria-hidden="true" /> </button>
Contrast
7 elements fail WCAG AA contrast ratio (4.5:1 minimum)
Specific failures found in your actual code:
• span.font-bold "NOW_SPINNING" in nav — text too light on background
• span.text-[8px] "LIVE" badge — white text on bg-primary at 8px is too small
• span.absolute "LV99" badge — white on primary color fails at small size
• span.text-[9px] text-gray-500 "CURATOR_ID" — gray-500 on light bg fails
• button.bg-[#FFEAEA] "LOGOUT" — red text on pink bg fails
• Search submit button — white text at text-[8px] on primary
• span.hidden sm:inline "ACTIVE SOURCE" text
Increase all text sizes below 12px to at least 12px — text-[8px] and text-[9px] are the main culprits.
For "CURATOR_ID": change text-gray-500 → text-gray-700 (dark mode: text-gray-300).
For "LOGOUT" button bg-[#FFEAEA]: use text-red-700 instead of lighter red to get 4.5:1.
Heading
h4 appears before any h2/h3 in the sidebar
Your sidebar has <h4 class="text-xs font-bold"> with no h2 or h3 parent. Screen readers rely on heading order to navigate. Change sidebar section labels to <h3> or <h2>.
Form
Volume range input has no label
Your <input type="range" min="0" max="100"> (volume slider) is invisible to screen readers — it has no <label> or aria-label. Fix:
<input type="range" min="0" max="100" aria-label="Volume" className="absolute inset-0 w-full opacity-0 cursor-pointer" />
Touch
Search submit button is 45×20 px — too small to tap
Google requires 24×24 px minimum tap target. Your search button is only 20 px tall. It's the button[type=submit] inside the search bar. Increase padding or min-height:
<!-- Before --> <button type="submit" className="absolute right-1 top-1/2 -translate-y-1/2 ...text-[8px]"> <!-- After: add min-h and bigger text --> <button type="submit" className="absolute right-1 top-1/2 -translate-y-1/2 ...text-xs min-h-[24px] px-2" aria-label="Search">
